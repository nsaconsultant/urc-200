## 2. URC-200 Protocol (`crates/urc200-proto`)

- [x] **S-010** — Table 11 operation commands encoder + `Freq` type with band/step validation. 26 unit tests + 5 proptests green.
- [x] **S-011** — Response parser + consecutive-NAK counter. **HIL-verified 2026-04-19** against real URC-200 V2. *Critical correction made during HIL:* ACK/NAK/HT terminators are the **ASCII control bytes 0x06 / 0x15 / 0x09**, not the 3-letter strings the manual appears to show. First-call-after-keypad `?01` yielded `41 31 09` (`A1` + HT); second call yielded `41 31 06` (`A1` + ACK). Parser updated and retested.
- [x] **S-012 (partial)** — Table 13 `Inquiry` enum with `?NN` encoder + typed decoders for `SynthLock`/`Rssi`/`PresetSnapshot`/`GeneralStatus`/`Mode`/`SquelchStatus`. **HIL-verified 2026-04-19** against real URC-200 V2:
  - `?01` → `A1` (synth locked) ✓
  - `?03` → `N122` (RSSI 122/255) ✓
  - `?10` → `T3083500R3083500M0N0C0P2#0` → decoded P2 @ 308.35 MHz AM simplex, low power ✓
  - `?11` → `X0J0L1d0F0v000` → PT / speaker off / lamp Lo / no options / temp OK ✓
  - Frequency math confirmed: 7 digits × 100 Hz = Hz value. `?10`'s 7th-digit is `0` here, so our S-010 command encoder (6 digits of kHz) will produce frequencies the radio accepts at 25/12.5 kHz grids. 5 kHz grid not verified yet — deferred until we attempt a fine-tuned frequency write.
  - Hammer's radio option byte confirmed `d0` = no EBN-30 / EBN-400 installed (matches his earlier FYI).
- [ ] **S-013** — MockTransport for CI *(belongs in `urc200-serial` crate alongside the `Transport` trait; landing in S-020)*

**Gate for Epic 3:** `cargo test -p urc200-proto` green with proptest fuzz. **CURRENTLY: 53 tests pass.**

**Hammer's radio config (recorded 2026-04-19):** base band only — VHF 115-173.995 MHz + UHF 225-399.995 MHz. No EBN-30 (LVHF), no EBN-400, no ECS-8 aviation option. The `Band::Lvhf` / `Band::Uhf400` types are still in the proto crate (near-free) to serve other URC users in the group who may have those options. ## Safety invariants (server-side — non-negotiable, per Murat)

- **Client heartbeat:** pings every 150 ms while PTT is asserted; server unkeys on 400 ms silence.
- **Socket close while owning PTT → immediate `E\r`.**
- **SIGTERM handler:** flush `E\r`, wait for serial drain (≤100 ms), exit. `docker stop` grace ≥ 5 s.
- **Startup invariant:** send `E\r` unconditionally before accepting any client connection (assume prior instance died keyed).
- **UAA `cpal::StreamError` → `E\r`** (unchanged from Tauri plan — Winston's original watchdog).
